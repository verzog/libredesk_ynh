## Overview

LibreDesk is a modern, open source, self-hosted customer support desk. It ships as a **single binary** backed by PostgreSQL and Redis — no Node, PHP, or Ruby required. The frontend is a Vue 3 single-page application served directly by the binary.

### Features

- **Multi shared inbox** – manage support@, billing@, sales@ and more from one place
- **Granular role-based permissions** – control exactly what each agent or team can see and do
- **Smart automation** – auto-tag, auto-assign, and auto-route conversations with custom rules
- **SLA tracking** – set response time targets and receive breach notifications
- **CSAT surveys** – automatically measure customer satisfaction
- **Macros** – one-click canned replies that can also set tags, change status, and more
- **Real-time updates** – WebSocket-powered live queue
- **AI-Assist** – rewrite replies for tone, clarity, or professionalism
- **Webhooks & API** – integrate with CRMs, analytics, or custom workflows

## Disclaimers / important information

### Dedicated domain required

LibreDesk **must** be installed on its own domain or subdomain (e.g. `helpdesk.example.com`). Installing it under a sub-path (e.g. `example.com/libredesk`) is **not supported** because LibreDesk's built-in SPA router needs to own the entire URL space.

### No LDAP / SSO

LibreDesk does not integrate with YunoHost's SSO portal or LDAP directory. Each LibreDesk user account must be created manually inside LibreDesk. The initial admin account uses the username **System**; set its password during the YunoHost install wizard.

### First-time setup after install

1. Open your browser and navigate to your domain.
2. Log in with email `System` and the password you chose during installation.
3. Go to **Settings → General** to configure your site name, logo, and timezone.
4. Create at least one **Inbox** (Settings → Inboxes) to start receiving conversations.
5. Invite your support agents (Settings → Agents).

### Supported architectures

Only **amd64** and **arm64** are supported. There are no 32-bit builds available upstream.
